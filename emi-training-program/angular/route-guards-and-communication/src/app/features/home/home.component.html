Route guard implementations:

  • CanActivate
    See the employees/create route in employees-routing.module.ts and auth.guard.ts.

  • CanDeactivate
    See the employees/create route in employees-routing.module.ts and create-employee-unsaved-changes.guard.ts.
    If you enter some data in the form and click "Cancel" the guard prevents routing.

  • CanLoad
    See the users route in app-routing.module.ts. The module will only load if you are an admin user.
    Check the console for "UsersModule loaded." message.

  • Resolve
    The users route does not resolve until the user data is loaded. See users-resolve.guard.ts.

Component communication implementations:

  • @Input()
    See employee-list.component.ts where the employee data is an input.
    Also see proficiency-details.component.ts where the selected employee is an input.

  • @Output()
    create-employee.compontent.ts emits the employee details to be saved and employee-list.component.ts emits the employee id number to be deleted.
    See proficiency-details.component.ts and loader.component.ts for more EventEmitters.

  • @ViewChild()
    create-employee.component.ts accesses the form in the template using the ViewChild decorator.

  • @ContentChild()
    proficiency-details.component.ts accesses the RatingComponent being projected in <ng-content> by using the ContentChild decorator.
